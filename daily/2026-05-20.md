## What I learned
- Context is the working state of an AI task: the instructions, history, constraints, and data the model can use right now.
- Good context is not just “more text”; it is the right text, in the right order, with clear priorities.
- Agent is an LLM plus a task loop and tools; it can plan, act, observe results, and continue, instead of only replying once.
- In Web3 scenarios, agent context should include network, wallet scope, contract address, cost limit, and confirmation rules.
- A safe agent should separate “analyze” from “execute”, especially for transactions, approvals, and irreversible actions.

## Key vocabulary
- Context: the information window the model uses to decide what to do next.
- Agent loop: plan -> act -> observe -> update -> continue.
- Tool permission: the actions an agent is allowed to take.
- Confirmation gate: a required user approval step before risky actions.
- State: the current working memory of the task, such as selected network, address, or objective.

## AI chat vs AI agent context

| Type | Example | Main difference |
| --- | --- | --- |
| Normal chat | “Explain what context means.” | One-off answer, no action loop. |
| Context-aware assistant | “Summarize these notes and keep the goal in mind.” | Uses given data and constraints. |
| Agent workflow | “Analyze this transaction, then wait for my approval before sending.” | Can plan and act, but must be gated. |

## My own explanation
Context is like the current桌面/working desk for the model: only the materials on the desk can be used immediately. If the desk is messy, the model may focus on the wrong thing. If the desk is organized, the model can reason more reliably. An agent is the same model operating with memory, tools, and a loop, so it can move from “answering” to “doing”.

## Simple Web3 agent flow
1. User gives a goal.
2. Agent reads context and identifies network, wallet, and contract.
3. Agent analyzes risk, cost, and possible side effects.
4. Agent shows a summary.
5. Agent waits for explicit confirmation.
6. Only after confirmation, agent executes the onchain action.
