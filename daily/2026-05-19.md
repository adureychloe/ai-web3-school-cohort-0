## What I learned
- LLM is the reasoning and language interface, but it does not automatically guarantee truth, safety, or execution correctness.
- Prompt is not magic wording; it is a structured task interface that should include goal, context, constraints, examples, and output format.
- In AI × Web3 workflows, prompt design must include permission boundaries, network choice, wallet/action scope, cost/risk disclosure, and confirmation requirements.
- WCB provides an Agent API entrypoint through `https://web3career.build/llms.txt`, with authenticated calls through `https://web3career.build/api/agent/call`.

## Key vocabulary
- LLM: a language model that can understand and generate text, code, plans, and structured outputs, but still needs grounding and verification.
- Prompt: the instruction layer that turns user intent into a specific task with context, constraints, and expected output.
- Tool permission: the boundary between what an agent can suggest and what it can actually execute.
- User confirmation: an explicit approval step before irreversible or risky actions, especially onchain transactions.
- Secret API Key: a private credential used to authenticate WCB Agent API requests; it must stay outside the public learning repo.

## Normal prompt vs Web3 agent prompt

| Prompt type | Example | Missing risk control |
| --- | --- | --- |
| Normal chat prompt | Explain what a wallet is. | Low risk; mostly informational. |
| Basic agent prompt | Help me interact with this contract. | Network, address, cost, permission, and confirmation may be unclear. |
| Safer Web3 agent prompt | Analyze this intended transaction, list network/address/cost/risk, and wait for my confirmation before execution. | Better boundary; still needs trusted tools and verification. |

## Safe Web3 Agent prompt template

```text
You are my AI × Web3 assistant. Before any onchain action, you must:
1. Identify the intended action.
2. Identify the network: mainnet, testnet, or local.
3. Identify contract/wallet addresses involved.
4. Estimate cost and irreversible risks.
5. Explain what data source or tool you used.
6. Output `requires_user_confirmation: true`.
7. Do not execute until I explicitly confirm.

Return:
- action:
- network:
- addresses:
- estimated_cost:
- risk:
- source:
- requires_user_confirmation:
- next_step:
```
