## What I learned

- AI × Web3 的交叉点不是“AI + 发币”或“聊天机器人 + 钱包”，而是把 AI 的理解/规划能力放进 Web3 的可验证、可限制、可结算机制里。
- Agent 一旦开始连接钱包、工具或付费服务，就必须考虑权限边界：能读什么、能写什么、能花多少钱、什么时候必须停下来让人确认。
- Chain-aware context 很关键。Agent 不能凭记忆判断链上状态，必须知道 chain id、地址、ABI、block number、交易哈希和数据来源。
- 我更感兴趣的方向是 Payment / Commerce / Settlement，因为它直接回答“Agent 如何在预算和规则边界内购买服务、完成交付、留下收据和可验证记录”。

## Key vocabulary

- Chain-aware Context
  - Meaning: 让 AI 在回答或行动前读取正确的链上事实，例如网络、地址、余额、授权、交易历史和区块高度。
  - Why it matters: 链上状态会变，错链/错地址/旧数据都可能导致危险建议。

- Agent Workflow
  - Meaning: 把用户目标拆成上下文读取、计划生成、工具调用、风险检查、人工确认、执行和复盘。
  - Why it matters: 高风险 agent 不能只靠一段 prompt，需要状态、边界和停止条件。

- Agent Wallet
  - Meaning: 不是把私钥交给 AI，而是给 agent 一个可限制、可撤销、可审计的行动空间。
  - Why it matters: 真实产品需要让 agent 能做事，但不能让它无限制控制资产。

- Machine Payment
  - Meaning: Agent、API、服务和钱包之间自动完成报价、授权、付款、收据和预算控制。
  - Why it matters: 没有可验证付款和预算控制，agent commerce 只能停在 demo。
